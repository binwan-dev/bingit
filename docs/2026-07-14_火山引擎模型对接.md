# 火山引擎模型对接支持

## 需求理解

当前项目仅支持单一 provider（DeepSeek），需要扩展为多 provider 架构，并新增火山引擎（Volcengine Ark）作为第二个默认 provider。

## 项目分析

### 当前架构

- `Config.provider: ProviderConfig` — 单 provider 配置
- `AiCommitConfig.model` — 格式为 `provider_name/model_name`，但验证时硬绑定到唯一的 provider
- `ai::generate_commit_message` — 直接从 `config.provider` 读取 api_base_url
- `resolve_api_key` — 从单 provider 或 `BINGIT_AI_KEY` 环境变量获取 key

### 火山引擎 API

- Base URL: `https://ark.cn-beijing.volces.com/api/v3`
- 兼容 OpenAI Chat Completions API (`/v1/chat/completions`)
- 常用模型: `deepseek-v4-pro`, `deepseek-v4-flash`

## 实现方案

### 1. 配置结构变更

将 `Config.provider: ProviderConfig` 改为 `Config.providers: Vec<ProviderConfig>`：

```json
{
  "AICommit": { "PromptPath": "...", "Model": "deepseek/deepseek-v4-flash" },
  "providers": [
    {
      "name": "deepseek",
      "ApiKey": "",
      "ApiBaseUrl": "https://api.deepseek.com",
      "models": ["deepseek-v4-flash"]
    },
    {
      "name": "volcengine",
      "ApiKey": "",
      "ApiBaseUrl": "https://ark.cn-beijing.volces.com/api/v3",
      "models": ["deepseek-v4-pro", "deepseek-v4-flash"]
    }
  ],
  "TokenLogging": false,
  "LogPath": "..."
}
```

### 2. 新增 `find_provider` 函数

根据 `provider_name` 从 `providers` 列表中查找匹配的 provider。

### 3. 修改 `validate_config`

从 providers 列表中按名称查找 provider，验证 model 在 provider 的 models 列表中。

### 4. 修改 `resolve_api_key`

接受 `&ProviderConfig` 参数而非 `&Config`。

### 5. 修改 `generate_commit_message`

接受 `&ProviderConfig` 参数，使用其 api_base_url。

### 6. 修改 `main.rs`

在 `run_ai_commit` 中先查找 provider，传递给 ai 调用。

## 涉及文件

- `src/config.rs` — 核心变更
- `src/ai.rs` — api 调用适配
- `src/main.rs` — 编排层适配

## 注意事项

- 字段名 `providers` 为小写，与现有 `provider` 风格一致
- 默认 provider 仍为 deepseek，保持向后兼容的默认行为
- 用户可通过配置文件手动切换 provider 或添加新 provider
- 火山引擎 API key 同样通过 `BINGIT_AI_KEY` 环境变量或 `ApiKey` 字段配置
